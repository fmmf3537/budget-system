针对资金计划功能，需要进行一些功能上的修改


## 最终验收规格（v4：单一口径、可实现/可测试/可验收）


### 1. 目标与范围
- 将“资金计划”升级为**按月度主计划 + 月度子计划**的协同编制与审批机制。
- 支持：
  - ADMIN 生成月度主计划（可选顶级部门范围）
  - 非 VIEWER 且 ACTIVE 的用户创建/提交子计划
  - 子计划提交后自动给主计划创建 ADMIN 审批
  - 子计划审批通过后计入主计划汇总（聚合式合并）
  - 余额/预测等敏感信息仅 ADMIN 可见（非 admin 接口不返回）
  - 主计划可删除且级联删除子计划/明细/审批记录

### 2. 名词定义
- **主计划**：MonthlyPlan。某组织某月的现金计划汇总容器；可选择一个“顶级部门范围”或全组织范围。
- **子计划**：SubPlan。归属某主计划，由业务用户提交的本月现金收支明细。
- **合并**：主计划汇总 = 其下所有 APPROVED 子计划明细的聚合结果（不做不可逆写入累加）。

### 3. 组织/部门口径（新增顶级部门范围）
- 主计划字段：`rootDepartmentCode: string | null`
  - `null` 表示全组织
  - 非空必须对应 **顶级部门**（BudgetDepartment 记录满足 `parentId = null`）
- 主计划禁止选择非顶级部门作为 root。
- 子计划字段：`scopeDepartmentCode: string`（若已有同义字段沿用）
  - 若主计划 `rootDepartmentCode = null`：子计划可选择任意存在的部门编码
  - 若主计划 `rootDepartmentCode = A`：子计划只能选择 A 子树内部门（建议允许选择 A 本身）
- 后端强制校验部门范围，前端仅做下拉过滤不作为安全边界。

### 4. 时间口径（本地日历月）
- 月度范围按本地日历月定义（例如 2026-04：4/1 00:00:00 ~ 4/30 23:59:59，本地）。
- 子计划明细行日期必须落在主计划月份范围内；否则保存/提交失败。

### 5. 金额规则（强校验）
- 子计划收入/支出明细金额：
  - 必须为 **正数**（> 0），不允许 0 或负数
  - 精度按系统货币/统一保留 2 位小数（实现时统一处理）

### 6. 用户范围与权限（验收口径）
- 可创建/编辑/提交子计划的用户需同时满足：
  - `User.status = ACTIVE`
  - 角色非 VIEWER（包括 ADMIN / BUDGET_MANAGER / APPROVER）
  - 最终建议用权限码实现（如 `CASH_PLAN_SUBPLAN_CREATE/EDIT/SUBMIT`），但验收时以“非 VIEWER 且 ACTIVE”作为最低要求。
- 主计划生成/删除/余额查看：仅 ADMIN（或具备对应权限）。

### 7. 状态机与操作

#### 7.1 主计划状态
- `OPEN`：允许创建/提交子计划
- `LOCKED`：禁止新建子计划与 DRAFT→SUBMITTED 提交（是否允许审批进行中的 SUBMITTED 继续审批：建议允许；若禁止需额外定义冻结规则）

#### 7.2 子计划状态
- `DRAFT`：可编辑（含增删明细）
- `SUBMITTED`：只读，等待审批
- `APPROVED`：已通过，计入主计划汇总
- `REJECTED`：驳回后回到 DRAFT（本项目验收口径：驳回动作使其回草稿可继续修改再提交）
- `WITHDRAWN/CANCELLED`：撤回提交（可选实现为回到 DRAFT 或单独状态；验收至少要求“撤回后可再次编辑并可重新提交”）

#### 7.3 操作规则
- **提交子计划**：DRAFT → SUBMITTED
  - 提交后立即为审批人生成待办（逐个子计划触发，不等待“全部提交”）
- **撤回子计划**：仅 SUBMITTED 可撤回
  - 撤回时取消待办：审批记录 PENDING → CANCELLED
  - 若审批链上已出现 APPROVED（有人同意过），则禁止撤回
- **审批同意**：SUBMITTED → APPROVED
- **审批驳回**：SUBMITTED → DRAFT（同时保留审批记录为 REJECTED）

### 8. 审批规则（固定给主计划创建 ADMIN）
- 子计划提交后，审批对象为“子计划”（entityType/entityId 指向子计划，不指向主计划）。
- 默认审批人 = 创建该主计划的 ADMIN 用户。
- 若该用户不再 ACTIVE / 不存在：由系统管理员接管
  - 接管口径：选择组织内 ADMIN 用户作为审批接收人（需明确选择策略：指定 superAdmin / 第一个 ADMIN / 配置项）

### 9. 主计划汇总与展示（合并口径）
- 主计划汇总 = 所有 APPROVED 子计划明细的聚合结果。
- 聚合口径（至少一套要固定）：
  - 按（方向 IN/OUT + 类别）汇总得到本月总流入/总流出/净流量
  - 如需日级预测：按（日期 + 方向 + 类别）进一步汇总
- 非 ADMIN 可见：
  - 子计划明细、流入/流出/净流量汇总
- 仅 ADMIN 可见（且非 admin 接口不返回字段）：
  - 期初余额、期末余额、预测余额曲线、安全水位、余额预警等

### 10. 删除规则（强约束）

#### 10.1 主计划删除（仅 ADMIN）
- 允许删除主计划（建议仅 OPEN 可删；LOCKED 需二次确认）
- 删除时级联删除：
  - 所有子计划及其明细
  - 所有关联审批记录（ApprovalRecord）

#### 10.2 子计划删除
- 建议仅 DRAFT 可删除；SUBMITTED/APPROVED 不允许直接删除（走撤回/驳回/作废流程）

### 11. 接口与页面（用于拆分任务）
- 主计划（ADMIN）：
  - 生成：选择月份 +（可选）顶级部门
  - 删除：可见危险操作按钮/二次确认
  - 详情：展示范围、子计划列表；ADMIN 额外显示余额区块
- 子计划（非 VIEWER active）：
  - 在主计划下创建：部门下拉按范围过滤
  - DRAFT 编辑/提交；SUBMITTED 撤回；被驳回后再编辑提交
- 审批（ADMIN/接管人）：
  - 待办按子计划维度展示；支持同意/驳回/转交（若系统支持）

### 12. 最小验收用例（必须全部通过）
1. ADMIN 生成 2026-04 主计划：同月重复生成同范围应提示“已存在”（不生成重复）。
2. 主计划 rootDepartmentCode 只能选顶级部门；选非顶级会被拒绝。
3. rootDepartmentCode = A 时：子计划部门只能选 A 子树（含 A 本身）；越界提交被拒绝。
4. rootDepartmentCode = null 时：子计划可选任意部门并成功提交。
5. 子计划明细日期超出当月：保存/提交失败。
6. 子计划金额为 0 或负数：保存/提交失败。
7. 子计划提交后：立即给主计划创建 ADMIN 生成待办（不等待“全部提交”）。
8. ADMIN 同意后：子计划变为 APPROVED，主计划汇总立即反映该子计划。
9. 非 ADMIN 查看主计划：接口不返回余额相关字段，但可见流入/流出/净流量汇总。
10. SUBMITTED 子计划可撤回且取消待办；若已出现 APPROVED 记录则禁止撤回。
11. 主计划删除：子计划、明细、审批记录被级联删除。

在月度计划详情页面里，非admin用户只能看到自己发起的子月度计划，不能看到全局数据。
